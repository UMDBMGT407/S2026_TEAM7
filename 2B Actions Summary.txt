ADMIN SIDE:

Dashboard: 
* Displays some static/sample data cards that are linked to different individual pages.
* The Upcoming Events card is linked to the View Events page, and the Shifts Needing Coverage page links to the Shift Management page. 
* There is a KPI Goals side pop-up which allows the admin to choose their weekly revenue targets, events this month target, and shifts filled target. There is a "set" button for each target that changes the color of the line below it, showing the target status. 

Event Requests Page: 
* Pending requests are connected to the customer event inquiry page and show how many customers have submitted requests that haven’t been accepted or rejected yet.
* Each event request has an accept and reject button. If a request is accepted, it moves to the View Events page and is saved in the database as “accepted,” along with all of the event details. If a request is rejected, it disappears from the website but is still saved in the database as “rejected.” Requests that haven’t been acted on are saved as “pending.
* The view events button navigates the user to the event details page that displays all of the accepted events 

View Events Page: 
* The View Events page shows all accepted events on one page with their full details. This page is connected to the event request page.
* Each accepted event has a “complete event” button. When this button is clicked, the event is marked as “complete” in the database and disappears from the page. * Even though it is no longer visible, it is still saved in the database as “complete.”
* There is also a button that links back to the event requests page.


Edit Promos: 
* Allows you to insert promotion information. 
* Allows you to apply promotions within a start date and end date, either for every day within that range or for a single recurring day within the range, depending on whether a recurring day is selected.
* The applicable menu items field is connected to the menu_item table and provides a dropdown of all available menu items.
* The Apply Promotion button applies the promotion to the promotion calendar, which is displayed on the View Promotions page.
* The View Promotions button navigates the user to the View Promotions page, where promotions are displayed on a calendar.


View Promos:
* The user can select from a dropdown menu of existing promotions and add them to the calendar. Promotions can be applied to every day within a selected date range or to a single recurring day within that range. Promotions are added by selecting the information and clicking the "Add to Calendar" button.
* The calendar allows navigation between months and years and is connected to a real calendar system, so all dates are accurate for each month.
* When a promotion is added to the calendar, it appears on the selected days or recurring days. Each promotion displays its title along with a view info button (to see full promotion details) and a delete button (to remove the promotion from the calendar).
* When a promotion is deleted from the calendar, it is removed from the promotion_calendar table but remains stored in the main promotions table in the database.

Staff Scheduling:
* Shows all scheduled admin and staff. 
* The user is able to click through the different schedule views (day, 3-day, week). 
* The user can filter through the different staff on the calendar.
* The calendar allows for toggling between previous and future days, along with a "Today" button to take the user to the current day.
* There is a red line following the calendar to show real time (EST).

Shift Management: 
* Allows the admin to create new shifts by selecting a staff member, role, day, and start/end time. When submitted, the shift is saved to the database and immediately appears on the calendar.
* Each scheduled shift includes a delete button, allowing the admin to remove shifts. Deleted shifts are removed from the schedule but remain handled through backend database logic.
* Includes a a “View Schedule” button that links to the full Staff Scheduling calendar page
* Displays a “Dropped Shifts & Requests” section where staff shift requests appear. The admin can accept or decline requests, which updates the request status in the database.

New User: 
* Allows the admin to create new users by entering information such as first name, last name, email, username, password, and role (admin or staff).
* The role selection determines what permissions the user will have within the system (e.g., admin access vs. staff access).
* User credentials (such as passwords) are securely handled and stored using hashing for authentication purposes.
* Newly created users can immediately log into the system and access pages based on their assigned role.
* Includes functionality to mark a user as inactive, which updates their status in the database without deleting their account.

Shift Availability: 
* Shows the signed-in staff their specific shifts for the week. 
* Allows the staff to select their own availability. 
* The users can drop their scheduled shifts or request a change in shift through the "Drop a Shift" and "Request Change" buttons. 
* They also get the same option through their "Actions" button in the "My Upcoming Shift" section. 
* This page is linked to the Shift Management page, as the staff's requests/availabilities will be shown on the Shift Management page.
